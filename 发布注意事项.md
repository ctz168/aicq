## 🚀 发布流程

### 自动发布（推荐）
```bash
# 1. 更新版本号
npm version patch  # 或 minor / major

# 2. 完整发布（构建 + GitHub Release）
npm run release

# 3. 仅发布到 npm（跳过构建）
npm run release:publish

# 4. 预览模式（不实际执行）
npm run release:dry
```

### 手动发布步骤

1. **构建所有平台**
   ```bash
   npm run install:all
   npm run build
   cd client/web && npm run build
   ```

2. **创建 Git Tag**
   ```bash
   git tag -a v{VERSION} -m "Release v{VERSION}"
   git push origin v{VERSION}
   ```

3. **创建 GitHub Release**
   ```bash
   # 使用 scripts/release.sh 或手动通过 GitHub API/网页创建
   GITHUB_TOKEN=your_token npm run release
   ```

4. **发布 npm 插件**
   ```bash
   npm config set //registry.npmjs.org/:_authToken=${NPM_TOKEN}
   cd plugin
   npm publish
   ```

## 📌 环境变量参考

| 变量 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `PORT` | 否 | `61018` | 服务器端口 |
| `DOMAIN` | 否 | `aicq.online` | 服务域名 |
| `JWT_SECRET` | **是** | (开发默认值) | JWT 签名密钥（生产必须设置） |
| `NODE_ENV` | 否 | - | 设为 `production` 启用生产模式 |
| `ALLOW_LOCALHOST` | 否 | `false` | 是否允许 localhost WebSocket 连接 |
| `CLICKHOUSE_URL` | **是** | `http://localhost:8123` | ClickHouse 连接 URL |
| `CLICKHOUSE_USER` | 否 | `default` | ClickHouse 用户名 |
| `CLICKHOUSE_PASSWORD` | 否 | (空) | ClickHouse 密码 |
| `CLICKHOUSE_DATABASE` | 否 | `aicq` | ClickHouse 数据库名 |
| `GITHUB_TOKEN` | 否 | - | GitHub API Token（发布用） |
| `NPM_TOKEN` | 否 | - | npm 发布 Token |
